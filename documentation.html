üîÑ Auto Version Detection

The system automatically checks version URLs every 5 minutes and extracts namespace and version information for both Production and Shadow environments.

How It Works

  1. Enter the Production URL (e.g., https://sportswidget.example.com/color.json)
  2. Click "Test URL" to verify it returns valid data
  3. Enter the Shadow URL for the preview environment
  4. Click "Test URL" to verify
  5. Save the component - versions will auto-update every 5 minutes

Expected JSON Format

The version URL should return JSON in this format:

JSON Field Example Value Extracted As
namespace mojito-example-green Namespace: green
staticImageTag 25.10.2.0-abc123 Version: 25.10.2.0
üí°
Automatic Status Change
If a version URL becomes unreachable, the component status automatically changes to POTENTIAL OUTAGE. It returns to Operational when the URL is reachable again.

Creating an Incident

  1. Go to Incidents tab
  2. Click "+ Create Incident"
  3. Fill in incident details (Title, Impact, Status)
  4. Select affected components
  5. Add summary, root cause, and resolution notes
  6. Click Save

Impact Levels

Level Description Use When
P1 Critical Widespread outage, major business impact
P2 Moderate Limited impact, non-critical features affected

  1. Click "ü§ñ Upload Details" button

  3. Select AI Provider (Groq or Google Gemini)
  4. Enter your API key (free from provider)
  5. Click "Process with AI"
  6. Review and accept/reject each suggestion
